按鈕會篩選下方卡片：訓練計算、驗證記錄、圖片輸出三條線彼此相連，但分析時最好分開看。

資料入口

  • SR 是 hazy 輸入，訓練時計算 (SR + 1) / 2 回到 [0,1]。
  • HR 是 diffusion 的 x_start，用於噪聲預測與 FFT loss。
  • depth 與 beta 來自 metadata，供 loss_t 使用。

netH 物理輸出

  • out_J 是去霧估計，out_T 是 transmission。
  • out_A 由 ANet(hazy) 預測，目前沒有直接 GT loss。
  • out_I = out_T * out_J + (1 - out_T) * out_A 重建 hazy。

反傳路徑

  • l_pix 回傳到 diffusion UNet。
  • 當 finetune_netH=true 且有物理目標時，loss_t/loss_asm 也參與反傳。
  • netH 只解凍指定 prefix，包括 conv_T_ 與 ANet.。

驗證 loss

  • validation 先採樣生成，再調用 update_physical_log()。
  • 記錄 loss_t、loss_asm、loss_physical_total 的 val set 平均。
  • validation 不重新計算 training 的 l_pix。

中間圖像

  • _out.png 是 diffusion 最終輸出。
  • _output.png、_out_T.png、_out_A.png、_out_I.png 用於分析 netH 物理約束。
  • _out_I.png 應與輸入 _lr.png 接近。

最容易誤解

  • l_pix 不是最終去霧圖對 GT 的普通 pixel L1。
  • loss_t 和 loss_asm 日誌值是未加權原始 L1。
  • 真正進總 loss 的是加權後 loss_physical_total。
Forward graph

當前代碼的 loss 資料流

主幹是 diffusion 噪聲預測；物理 loss 是附加在 netH 中間量上的輔助約束。

1

Hazy input

SR 從 [-1,1] 轉為 hazy_input_01，送入 MPRfusion。

2

netH outputs

得到 out_J、stage1_output、out_T、out_A、out_I。

3

Condition

cat([out_J/0.5-1, out_T/0.5-1]) 形成 4 通道條件。

4

Diffusion loss

UNet 輸入為 condition 4 通道加 noisy HR 3 通道，共 in_channel=7。

5

Total loss

l_pix 加權合併 loss_t 與 loss_asm 後反傳。

Loss ledger

三個主要 loss 的角色分工

下方圖表用文檔中的範例日誌數值計算：l_pix=0.52307、loss_t=0.13905、loss_asm=0.077069。

l_pix

Diffusion 噪聲預測 + FFT 幅值約束

L1_sum(noise, pred_noise) + 0.01 * L1_sum(abs(FFT(x0_pred)), abs(FFT(HR)))，再除以 B*C*H*W。

權重 1.0，主導 total loss
loss_t

Transmission map 物理監督

out_T 對 t_gt = exp(-beta * depth) 的 mean L1，兩者都 clamp 到 [1e-4, 1.0]。

加進總 loss 前乘 0.01
loss_asm

大氣散射模型重建一致性

out_I 對輸入 hazy 圖 hazy_input_01 的 mean L1，不需要額外 atmospheric light GT。

加進總 loss 前乘 0.05

加權後對 loss_total 的貢獻

物理項在數值上較小：0.01*loss_t + 0.05*loss_asm = 0.005244，主要用來牽引 netH 中間物理量。

Physical equations

物理約束到底約束誰

out_A 沒有獨立 supervision；它主要靠 out_I 的重建一致性被間接調整。

Transmission target

Beer-Lambert / scattering prior
t_gt(x) = exp(-beta * depth(x))

若 depth 尺寸不同，會 bilinear resize 到 out_T 尺寸；若 batch beta 是 scalar，會 expand 到 batch 維度。

Atmospheric scattering reconstruction

Self-consistency
out_I = out_T * out_J + (1 - out_T) * out_A

這裡的目標不是 GT clear，而是原始 hazy 輸入 (SR + 1) / 2。

Code anchors

主要代碼位置

這些位置對應目前 HTML 中每一塊設計判斷，可直接回源碼驗證。

model/model.pyloss 組合
  • optimize_parameters()：netH forward、condition 拼接、l_pix、物理 loss、total_loss.backward()。
  • _compute_physical_losses()：loss_t 與 loss_asm 的實作。
  • update_physical_log()：validation 階段更新物理指標。
model/networkHelper.pynetH 輸出
  • out_J、stage1_output、out_T、out_A 都 clamp 到 [0,1]。
  • out_I 按大氣散射公式合成。
model/sr3_modules/diffusion.pyl_pix 來源
  • p_losses() 對 noise prediction 做 L1。
  • 再對 x0_recon 與 HR 的 FFT 幅值加 0.01 權重。
config/...physical.json實驗設定
  • lambda_t=0.01、lambda_asm=0.05。
  • finetune_netH=true、in_channel=7、validation 使用 DDIM 20 steps。
分析重點

loss_total 仍由 diffusion 訓練 loss 主導；physical loss 的價值在於用 out_T 的深度先驗與 out_I 的散射重建一致性，讓 netH 的中間物理量更可解釋、更穩定。
